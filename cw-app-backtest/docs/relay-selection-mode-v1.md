# 连板接力选股模式 V1

> 业务状态：已确认
> 确认日期：2026-07-22
> 实现状态：已与选股代码和日 K 理论结果研究流程对齐
> 范围：定义连板接力的选股触发、候选板数、选股强度、排序、数量，以及用于收敛选股参数的日 K 理论结果口径；不改动正式买入、卖出和持仓执行规则。

## 1. 文档定位

本文是连板接力选股和分场景回测的业务口径。

后续实现、回测或页面展示涉及连板接力模式时，应先读取本文，再检查生产代码与本文的差异。统一收盘后选股入口现已按固定顺序启用连板接力、普通首板和小市值首板；页面“重新选股”和收盘调度都会执行三个模式。连板研究接口仍单独保存全候选和理论结果。

## 2. 核心概念

### 2.1 高度身份

- 股票达到 7 板，称为龙头。
- 股票达到 9 板，称为妖股。
- 股票达到 11 板，称为大帝。
- 股票从 7 板开始占领情绪周期。

### 2.2 情绪周期占领

股票达到连板高度 `H` 后，占领数量为：

```text
占领数量 = H - 3
```

| 已完成连板高度 | 占领数量 |
|---:|---:|
| 7 板 | 4 个 |
| 8 板 | 5 个 |
| 9 板 | 6 个 |
| 10 板 | 7 个 |
| 11 板 | 8 个 |

后续股票没有超过当前占领者的历史最高连板高度时，只能算跟风，跟风股断板不能触发高切低。

未达到妖股高度前，新股票在当日收盘封死的连板高度超过当前占领者的历史最高连板高度时，当天立即接管情绪周期：

```text
新股票当日封死连板高度 > 当前占领者历史最高连板高度
```

例如，前一占领者最高完成 7 板并在 7 进 8 时断板，后续股票完成 7 板时仍然是跟风；后续股票封死 8 板时，当天立即接管情绪周期。

当新股票收盘封死 9 板或更高高度时，已经进入妖股范畴，无论前一占领者的高度是多少，都在当天直接接管情绪周期。它能够到达 9 板，说明没有继续受到前一周期压制，市场做多接力情绪良好，不触发高切低。

### 2.3 断板定义

断板只看收盘是否涨停：

- 盘中炸板但收盘重新封死涨停，不算断板；
- 收盘没有封住涨停，算断板。

只有当前情绪周期占领者断板，才触发高切低。

## 3. 触发体系与优先级

连板接力分为三类触发：

1. 高度压制；
2. 高切低；
3. 卡位。

主选股链使用互斥的 `if / else if`；卡位在主链之后单独判断：

```java
if (命中高度压制) {
    // 高度压制选股
} else if (命中高切低第二次补充) {
    // 先完成前一日已经确认的断板事件
} else if (命中高切低断板当天) {
    // 再判断当日是否发生新的占领者断板
}

if (前面没有选出候选 && 命中弱唯一5板) {
    // 最后的卡位选股
}
```

主链顺序必须保持为“高度压制 -> 高切低第二次补充 -> 高切低断板当天”。断板第二次补充放在断板当天之前，是为了避免连续退潮时两个高度条件同时成立，从而把同一次断板重复识别成新的断板。

## 4. 高度压制

### 4.1 触发条件

只要当日市场最高连板高度为 2、3、4 板，就属于高度压制，不要求相对前一交易日下降。

以下情况都属于高度压制：

- 昨日 3 板，今日仍为 3 板；
- 昨日 2 板，今日升到 3 板；
- 昨日 4 板，今日降到 3 板；
- 昨日 3 板，今日升到 4 板。

当日市场最高板达到 5 板及以上时，才退出高度压制分支。

### 4.2 候选板数、强度和数量

| 当日市场最高板 | 候选板数 | 选股强度 | 数量上限 |
|---:|---:|---|---:|
| 2 板 | 2 板 | 宽松 | 3 只 |
| 3 板 | 3 板 | 宽松 | 与 2 板合计 3 只 |
| 3 板 | 2 板 | 一般 | 与 3 板合计 3 只 |
| 4 板 | 3 板 | 宽松 | 与 2 板合计 3 只 |
| 4 板 | 2 板 | 一般 | 与 3 板合计 3 只 |

## 5. 高切低

### 5.1 占领者断板当天

断板来源高度为 5 板或 6 板时，只要下一交易日最高高度没有继续突破，就可以触发高切低，不要求核对占领者，因为市场连 7 板龙头高度都无法达到，低一位股票继续晋级的概率也受到压制。

断板来源高度达到 7 板及以上时，必须确认断板股票是该参考日的情绪周期占领者，才触发第一次高切低。跟风高位股断板不触发。

| 候选板数 | 选股强度 | 数量上限 |
|---:|---|---:|
| 3 板 | 严谨 | 与 2 板合计 3 只 |
| 2 板 | 严谨 | 与 3 板合计 3 只 |

时间口径：

```text
D 日收盘确认占领者断板
-> D 日收盘执行第一次高切低选股
-> 候选用于 D+1 交易
```

### 5.2 断板后的第二次补充

第二次补充的触发条件为：

```text
前一交易日已经满足同一套断板来源规则
并且当前交易日没有形成新的有效接管
```

| 候选板数 | 选股强度 | 数量上限 |
|---:|---|---:|
| 3 板 | 一般 | 3 只 |

时间口径：

```text
D 日收盘：占领者断板，执行第一次高切低选股
D+1 收盘：执行第二次补充选股，候选用于 D+2 交易
```

第二次补充直接执行选股，不以前一次是否已经买入作为前置条件。前一次没有买到，或者买入后炸板并快速卖出，属于后续买卖执行规则，不改变第二次选股本身。

如果 D+1 有新股票超过前占领者的历史最高高度并封死涨停，情绪周期占领代码发生变化，则取消第二次补充。新的占领者说明市场接力情绪良好、没有产生高切低所需要的分歧。

## 6. 卡位

卡位是主选股链之后的独立兜底判断。

### 6.1 触发前提

必须同时满足：

1. 主选股链没有产生候选；
2. 当日市场最高板为 5 板；
3. 触发日前 10 个交易日的市场最高板算术平均值严格 `<6`；
4. 过滤 ST 后恰好只有一只 5 板股票；
5. 唯一 5 板的质量指标完整；
6. 唯一 5 板命中至少一项弱势条件。

前 10 个交易日不包含当日，从情绪周期表按交易日倒序取前 10 条完整数据。
平均值等于 `6.0` 或大于 `6.0` 时，直接不执行弱唯一 5 板逻辑；不足 10 个交易日，
或任一交易日的市场最高板数据缺失时，同样不执行。该条件只约束卡位兜底，
不改变高度压制、高切低和高切低第二次补充的触发结果。

### 6.2 弱唯一 5 板条件

以下任意一项成立即视为弱：

| 指标 | 触发条件 |
|---|---:|
| 当日流通市值 | `>45亿元` |
| 当日换手率 | `>45%` |
| 当日振幅过大 | `>13%` |

边界值 45 亿元、45% 和 13% 本身不触发弱势判断。一字板、低振幅和本轮启动价格不再作为弱唯一 5 板的判断条件。

### 6.3 卡位选股

| 候选板数 | 选股强度 | 数量上限 |
|---:|---|---:|
| 2 板 | 严谨 | 3 只 |

## 7. 三级选股强度

三种强度使用同一套评分公式。强度只影响过滤条件和阈值，评分只负责同板候选排序，不设最低评分门槛。

| 条件 | 严谨 | 一般 | 宽松 |
|---|---:|---:|---:|
| 启动流通市值 | `<20亿元` | `<35亿元` | `<50亿元` |
| 最近 200 根有效 K 线历史最大换手率 | `<=40%` | `<=45%` | `<=50%` |
| 启动价格 | `>3元` | 不限制 | 不限制 |
| 最低评分 | 不设 | 不设 | 不设 |
| 完整筹码判断 | 执行 | 执行 | 执行 |
| 其他共同过滤 | 暂时沿用现有规则 | 暂时沿用现有规则 | 暂时沿用现有规则 |

启动流通市值和启动价格沿用现有口径，取本轮首板前一个交易日收盘时的数据。历史最大换手率取本轮启动前最近 200 根有效日 K 的最大换手率。

V1 阶段除表中三项差异外，前 20 日异常、近期形态、90 日历史指标以及其余完整筹码判断使用同一套规则。校准样本回测后，加速缩量和近期形态采用以下边界：

- 2 板出现两根加速缩量板时，只有最近 200 根有效 K 线历史最大换手率 `<20%` 才继续执行其余完整过滤；
- 3 板出现两根加速缩量板时继续否决，避免放宽项优先占用 3 板排序位置；
- 2 板包含选股日在内的 5 日复权振幅要求 `<35%`；
- 3 板包含选股日在内的 5 日复权振幅仍要求 `<=48%`。

完整筹码阶梯增加两条校准通道：

- 启动流通市值 `>20亿元且<=30亿元`、历史最大换手率 `>=40%且<46%`、选股日价格 `<10元`；
- 启动流通市值 `>25亿元且<=30亿元`、历史最大换手率 `>=25%且<30%`、选股日价格 `>=16元且<25元`。

以上通道只扩展完整筹码判断，不绕过 ST、可转债、涨幅、当日价格、强度市值/换手、前 20 日形态和 90 日历史指标等前置条件。

## 8. 排序与数量截断

所有同时包含 2 板和 3 板候选的场景统一执行：

1. 连板高度降序，3 板优先于 2 板；
2. 同板内按选股分数降序；
3. 同板同分时，按选股当日价格升序；
4. 价格仍相同时，按股票代码升序；
5. 最终截取前 3 只。

如果合格 3 板票已经达到 3 只，则全部选择 3 板票，2 板票不再入选；3 板不足 3 只时，才使用 2 板票补足。

## 9. V1 回测原则

### 9.1 原始候选池

- 每个触发日先按该触发点需要的板数建立原始池，再执行 ST、可转债、市值、价格、筹码和近 20 日形态等过滤；
- 高度 2 只记录 2 板；高度 3/4 记录 2、3 板；高切低当天记录 2、3 板；第二次补充只记录 3 板；卡位只记录 2 板；
- 原始池中的每只股票都保存选股日快照、辅助指标、第一拒绝原因、板内名次、最终名次和是否进入 Top3；
- 没有合格候选时允许空仓，不设置最低样本数，也不强制补足 3 只。

### 9.2 D+1 打板与观察终点

- D 为推荐/选股日；D+1 为下一交易日；
- 只做打板，理论买入成本固定为 D+1 涨停价；
- D+1 `kline_state > 0` 视为触板，所有触板候选都视为理论买入；未触板只保留样本，不发生理论交易；
- 已触板样本直接使用 D+1 日 K 最高价作为实际涨停价，避免固定比例公式在除权、ST、退市整理或不同板块上产生偏差；
- 候选在精确 D+1 没有日 K 时标记为数据缺失，不能把其恢复交易后的第一根日 K 当成 D+1；
- 收盘涨停状态为 `1` 至 `5`；盘中炸板后回封继续持有；
- 从 D+1 起观察到第一次收盘不涨停的交易日，该日为断板日，并以断板日最高价计算理论最大收益；
- D+1 触板但收盘炸板时，断板日就是 D+1，理论最大毛收益记为 0；
- 数据截止日仍收盘涨停的样本标记为未完成，不进入完整收益和胜率分母。

### 9.3 指标优先级

1. 封板率：分母为 D+1 触板候选，分子为 D+1 收盘封板候选；
2. 理论胜率：所有已完成的触板理论交易中，断板日理论最大收益大于 0 的比例；
3. 理论最大收益：按触发、板数、强度和参数组合比较；
4. 每个触发日从全原始池 D+1 触板且结果完整的股票中找出理论最优股票，统计正式 Top3 是否捕获，以及捕获的收益比例。

首轮拟合区间为 `2025-01-01` 至 `2026-07-21`；后续使用 `2023-01-01` 至 `2024-12-31` 做区间外验证。允许后续回测调整筹码、市值、价格和近 20 日形态规则；触发优先级、板数计划、三级强度归属、Top3 排序、D+1 涨停价成本和首次收盘断板终点保持冻结。

## 10. 研究接口

```text
POST /backtest/relay-selection-runs
     ?startDate=2025-01-01
     &endDate=2026-07-21

GET  /backtest/relay-selection-runs
GET  /backtest/relay-selection-runs/{runId}
GET  /backtest/relay-selection-runs/{runId}/triggers
GET  /backtest/relay-selection-runs/{runId}/candidates
     ?triggerRecordId={可选}
     &page=1&pageSize=100
```

运行、逐日触发和逐股候选分别落在：

- `relay_selection_run`；
- `relay_selection_trigger_record`；
- `relay_selection_candidate_record`。

研究任务还会把每个推荐日最终入选的 Top3 同步写入 `stock_watching_task`，供回测系统
沿用原有任务读取链路。写入时固定使用 `trade_mode=1`，并保存 `selection_run_id` 和
`relay_candidate_record_id`，可以从任务追溯到本轮研究及对应的全候选审计记录。

同一推荐日重跑时，只删除并替换该日的 mode 1 任务，不影响普通首板和小市值首板；
当日没有触发或没有最终入选候选时，也执行空替换，避免保留旧规则产生的连板任务。
因此 `stock_watching_task` 保存重叠日期最近一次研究运行的最终结果，三张研究表继续承担
按 run 永久保存全候选、过滤轨迹和理论结果的职责。
